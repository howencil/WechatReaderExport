# 微信读书笔记导出器 📚

> 一键导出微信读书笔记，支持Markdown、JSON、TXT多种格式，让你的读书笔记永不丢失！

## ✨ 功能特性

### 🎯 智能识别
- **自动检测页面类型** - 智能识别微信读书不同页面，准确判断是否可导出
- **书籍信息提取** - 自动获取书名、作者、封面等详细信息
- **笔记统计** - 实时显示当前页面的笔记数量

### 📝 强大提取
- **多种笔记类型** - 支持高亮标注、个人想法、书签等所有笔记类型
- **完整内容** - 提取原文摘录、个人笔记、章节信息、创建时间等完整数据
- **智能识别** - 即使页面结构变化，也能准确识别笔记内容

### 📄 多格式导出
- **Markdown格式** - 按章节整理，支持引用格式，便于后续编辑
- **JSON格式** - 结构化数据，适合程序处理和数据分析
- **TXT格式** - 纯文本格式，兼容所有设备和软件

### 💫 极致体验
- **一键操作** - 简单三步完成导出：检测→选择→下载
- **实时反馈** - 清晰的状态提示和进度显示
- **智能命名** - 自动生成包含书名和时间的文件名

## 🚀 快速开始

### 安装方式

#### 方法一：Chrome Web Store（推荐）
*即将上线，敬请期待...*

#### 方法二：手动安装
1. 下载插件包：[点击下载](../../releases/latest)
2. 解压到本地文件夹
3. 打开Chrome浏览器，访问 `chrome://extensions/`
4. 开启右上角的"开发者模式"
5. 点击"加载已解压的扩展程序"，选择解压的文件夹
6. 安装完成！插件图标会出现在浏览器工具栏

### 使用方法

#### 第一步：打开微信读书
访问 [微信读书官网](https://weread.qq.com)，登录你的账号

#### 第二步：进入笔记页面
- **方式一**：选择任意书籍 → 点击"笔记"标签
- **方式二**：在阅读页面直接使用（会导出当前阅读的笔记）

#### 第三步：使用插件导出
1. 点击浏览器工具栏中的插件图标
2. 确认显示绿色状态（如"检测到笔记页面，发现10条笔记"）
3. 选择导出格式（推荐Markdown）
4. 点击"导出笔记"按钮
5. 在弹出的保存对话框中选择保存位置
6. 完成！

## 📋 支持的页面

| 页面类型 | 支持状态 | 说明 |
|---------|---------|------|
| 笔记页面 | ✅ 完全支持 | 专门的笔记查看页面，推荐使用 |
| 阅读页面 | ✅ 完全支持 | 在阅读过程中直接导出当前笔记 |

## 📊 导出格式说明

### Markdown格式（.md）
**推荐用途**：个人笔记整理、博客写作、知识管理

**格式特点**：
- 按章节自动分类整理
- 原文使用引用格式显示
- 包含完整的书籍信息和时间戳
- 兼容所有Markdown编辑器

**示例输出**：
```markdown
# 《思考，快与慢》

**作者:** 丹尼尔·卡尼曼
**导出时间:** 2025-09-07 20:30:15
**笔记数量:** 15 条

---

## 第一章 一张心智地图

### 笔记 1
> 我们的大脑有快与慢两套决策系统...

**我的想法:** 这个概念很有趣，解释了很多日常决策的偏差

*创建时间: 2025-09-07 15:30:00*
```

### JSON格式（.json）
**推荐用途**：数据分析、程序处理、备份存档

**格式特点**：
- 完整的结构化数据
- 包含所有元数据信息
- 便于程序读取和处理
- 支持数据迁移和转换

### TXT格式（.txt）
**推荐用途**：简单查看、打印输出、设备兼容

**格式特点**：
- 纯文本格式，兼容性最佳
- 清晰的排版结构
- 支持所有设备和软件
- 文件体积最小

## 🔧 常见问题

### Q: 为什么检测不到笔记？
**A:** 请确保：
1. 当前页面确实包含笔记内容
2. 页面已完全加载完成
3. 尝试点击刷新按钮（🔄）重新检测
4. 刷新微信读书页面后重试

### Q: 导出的文件在哪里？
**A:** 文件保存在浏览器的默认下载文件夹中，文件名格式为`书名_笔记_日期.格式`

### Q: 能否批量导出多本书的笔记？
**A:** 目前版本暂不支持批量导出，需要逐本操作。批量功能已在开发计划中。

### Q: 导出的笔记不完整怎么办？
**A:** 
1. 确认页面显示了所有笔记（可能需要滚动到底部加载）
2. 尝试刷新页面后重新导出
3. 检查是否存在分页，需要手动翻页后再导出

### Q: 插件安全吗？
**A:** 
- 插件完全本地处理，不会上传任何数据到服务器
- 开源代码，可审查安全性
- 只请求必要的浏览器权限
- 符合Chrome Web Store安全标准
